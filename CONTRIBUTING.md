# Contributing to Cybersec Connect

First off, thank you for considering contributing to Cybersec Connect! It's people like you that make this such a great community.

## Where do I go from here?

If you've noticed a bug or have a feature request, please [open an issue](https://github.com/Kyle6012/g24sec-space-legacy/issues/new). It's generally best to discuss your ideas in an issue before making a pull request.

## Fork & create a branch

If you've decided to contribute, the first step is to fork the repository and create a new branch for your changes.

```bash
git clone https://github.com/Kyle6012/g24sec-space-legacy.git
cd g24sec-space-legacy
git checkout -b my-new-feature
```

## Making your changes

Make your changes to the codebase, and then commit them with a descriptive commit message.

```bash
git add .
git commit -m "feat: add my new feature"
```

## Submitting a pull request

Once you're happy with your changes, push them to your fork and then open a pull request. We'll review your changes as soon as we can.

```bash
git push origin my-new-feature
```

Thank you for your contribution!
